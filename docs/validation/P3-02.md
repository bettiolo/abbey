# P3-02 — Seed-slot settlement growth + Asylum Corner rename

## What changed

A new layer under `Runtime/Settlement/` turns free building placement into clustered
seed-slot growth (ROADMAP Phase 3 item 14). `SeedSlotSystem` (a singleton like
`SeasonSystem`) holds a registry of `SeedSlot`s — each with a position, size class and
`Locked`/`Open`/`Occupied` state. `BuildingPlacer` now refuses placement unless the
position falls on an `Open` slot (new `PlacementError.NoOpenSlot`), snapping to and
occupying that slot; when **no** `SeedSlotSystem` is in the scene, placement stays
unconstrained (Phase 2 behaviour, and how the fixed abbey `RestorationNode`s bypass
slots). Completing a building fires the new static `Building.Constructed` signal, and
`SeedSlotSystem.OpenChildSlots` lays child slots on a ring around it — keeping only
those that **hug** (touch an existing building or sit on lit ground per
`DarknessEvaluator`), so the village stays compact. `ComputeLightDebt()` sums the
ground area of slots/buildings sitting outside Safe light, weighted per zone — the
overextension penalty. All tunables live in the `SettlementGrowthConfig`
ScriptableObject (Resources path `SettlementGrowthConfig`, `CreateInstance` defaults
like `EconomyConfig` — no asset file required). Hidden state is shown on the new
**F8 SettlementDebugPanel** (slot counts, light debt, per-slot gizmos).

Phase 2 cleanup in the same task: the legacy `infirmary_corner_t1` building is renamed
to **`asylum_corner_t1`** everywhere — asset spec + regenerated glb/previews/metadata,
`BuildingCatalog` id, `FunctionKind.Infirmary → Asylum`,
`RestorationNodeKind.InfirmaryCorner → AsylumCorner`,
`AbbeyState.InfirmaryBuilt → AsylumBuilt`, and `InfirmaryZone → AsylumZone`. The
`AsylumZone` keeps the trigger radius (from `PrototypeConfig.asylumRadius`) and tracks
occupancy, but the Phase 2 instant `ForceState(Idle)` heal is **removed** — it is a
documented shell that P3-03 fills with sanity recovery.

Public API downstream tasks consume: `SeedSlotSystem.Instance`
(`Slots`, `AddAuthoredSlot`, `FindOpenSlotNear`, `OccupySlot`, `OpenChildSlots`,
`ComputeLightDebt`, `CountByState`), the `SeedSlot`/`SlotState`/`SlotSizeClass` types,
`Building.Constructed`, and the `AsylumZone` shell (`radius`, `Occupants`).
(P3-03 sanity/asylum hangs recovery off `AsylumZone`; P3-12 desire paths and P3-13
island exploration grow the map through the seed-slot API.)

## Visual verification

1. `Tools → Abbey → Build Prototype Scene`, then press Play.
2. Press **F8** to open the SettlementDebugPanel (bottom-right, below the F7 season
   panel). It lists slot counts by state, the live light debt, and the first slots.
   In the Scene view the slot gizmos ring the camp: **green** = Open, **amber** =
   Occupied, **grey** = Locked.
   - **Correct:** several green Open gizmos hug the campfire / shelters and the
     lantern-lit path; one grey Locked plot sits further out.
   - **Failure:** no gizmos appear, or "no SeedSlotSystem in scene".
3. Press **F3** to open the BuildingDebugPanel and move the placement marker (the
   hero) around, watching the per-catalog place-ability:
   - **Correct:** `place` is enabled only when the marker sits on an Open slot;
     off-slot the row shows `[NoOpenSlot]` and the button is disabled. Placing on a
     green slot flips it **amber** (Occupied) in the F8 gizmos.
   - **Failure:** you can place a building anywhere, or on-slot placement is refused.
4. Complete a building on an occupied slot (deliver its materials and pump work from
   the F3 panel):
   - **Correct:** on completion **new green Open gizmos pop up beside it** (child
     slots), and the F8 light-debt figure rises if any new slot is in the dark and
     falls again once a lantern/campfire covers it.
   - **Failure:** completing a building opens no child slots, or opens slots stranded
     in the dark with no light or building beside them (the hug rule is broken).
5. Asylum corner: build the abbey `AsylumCorner` restoration node (F3 / abbey hill).
   - **Correct:** it completes as an **Asylum Corner** carrying an `AsylumZone`
     (select it to see the purple radius gizmo); villagers standing inside are tracked
     as occupants but are **not** instantly healed (that is P3-03). The abbey log shows
     `asylum_built`, never `infirmary_*`.
   - The regenerated art preview renders under
     `blender/generated/previews/asylum_corner_t1_preview_*.png`.
