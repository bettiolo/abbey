# P3-01 — Seasonal calendar, weather and light modifiers

## What changed

A new simulation layer under `Runtime/World/` turns the year and rolls the sky.
`SeasonSystem` observes the `GameClock` day counter (via the new
`EventBus.DayChanged` callback) and derives the current season — Spring (hope) →
Summer (growth) → Autumn (warning) → Winter (judgment) — from `SeasonConfig`; it
scales the clock's **night phase** by a per-season multiplier so nights lengthen
toward Winter. `WeatherSystem` draws a deterministic, seed-stable weather state
(clear/fog/rain/tempest), reads the moon phase, and rolls the White Night omen for
each calendar day, then publishes two effectiveness multipliers: a **global light
multiplier** into `DarknessEvaluator.LightEffectivenessMultiplier` (scales every lit
radius) and a **bell-reliability multiplier** into
`BellkeeperController.BellReliabilityMultiplier` (shrinks the recall pulse). All
tunables live in the `SeasonConfig` ScriptableObject (Resources path `SeasonConfig`,
`CreateInstance` defaults like `EconomyConfig` — no asset file required). Hidden
state is shown on the new **F7 SeasonDebugPanel**.

Public API downstream tasks consume: `SeasonSystem.Instance`
(`CurrentSeason`, `DayOfYear`, `DayInSeason`, `YearNumber`, `NightLengthMultiplier`)
and `WeatherSystem.Instance` (`CurrentWeather`, `CurrentMoonPhase`, `IsWhiteNight`,
`LightEffectivenessMultiplier`, `BellReliabilityMultiplier`, `NextWhiteNightDay()`),
plus `EventBus.SeasonChanged/WeatherChanged/OmenAppeared/DayChanged`.

## Visual verification

1. `Tools → Abbey → Build Prototype Scene`, then press Play.
2. Press **F7** to open the SeasonDebugPanel (bottom-right). It shows season, year,
   day-in-season, day-of-year, the night-length multiplier and the resulting night
   seconds, plus weather, moon phase, the live light/bell multipliers and the next
   omen day.
3. Scrub time faster (lower the day/dusk/night/dawn seconds in the `PrototypeConfig`
   asset, or let it run) and watch across several in-game days:
   - **Correct:** the season label advances Spring → Summer → Autumn → Winter; the
     reported night length **grows** toward Winter (winter ≈ 1.6× the spring night);
     `SeasonChanged` / `WeatherChanged` / `OmenAppeared` records appear in the F1
     event log.
   - **Failure:** the season never changes, or the night-second figure stays constant
     as seasons turn.
4. Force a tempest to see light degrade: in the `SeasonConfig` (create one via
   `Assets → Create → Abbey → Season Config` at `Resources/SeasonConfig`) set every
   season's weather weights to tempest-only, or just watch a naturally-drawn tempest
   in the panel.
   - **Correct:** under a tempest the panel's `light x…` and `bell x…` values drop
     below 1.0, the `LightZoneGizmos` yellow/orange rings visibly **shrink** around
     campfire/lantern/abbey flame, and villagers that were Safe near the fire's edge
     slip toward Edge/Dark. On a full moon the rings swell slightly (light > 1.0).
   - **Failure:** the gizmo rings and multipliers never move with the weather label,
     or a tempest leaves the bell recall reaching just as far (compare the
     `hero_rang_bell radius=…` log line under clear vs tempest — it must be smaller
     under tempest).
