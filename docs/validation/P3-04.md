## What changed

Phase 2's economy was salvage-only (a finite wreck). P3-04 adds a **renewable,
seasonal** production layer so the settlement can survive a full year â€” and pays for
it with **Winter = judgment**: growth halts in winter, forcing stockpiling.

- **Resources.** `ResourceType` gains six values (indices 8..13, appended so serialized
  indices stay stable): `Grain, Meat, Wool, Herbs, Tools, Coal`. Grain is the raw
  harvest; `Food` stays the eat-ledger (grain mills to food at
  `EconomyConfig.grainToFoodRatio`, a downstream conversion, not an automatic ledger
  effect). They round-trip through `ResourceLedger` unchanged.
- **Production buildings.** New `Runtime/Economy/ProductionBuilding.cs` component (a
  registry sibling of `SalvageSite`) runs a data-driven `ProductionRecipe` each in-game
  day: it adds one worker-day per staffed worker, and when the cycle length is reached
  it consumes recipe inputs and deposits outputs to the ledger (event-logged
  `production` + `resource` records). **Growth** recipes (field, pasture) scale their
  yield by the season and **do not advance in winter** (yield 0 halts them);
  **conversion** recipes (charcoal kiln â†’ coal, smithy â†’ tools) run year-round at Ă—1.
  New `FunctionKind.Production` buildables in `BuildingCatalog`: `field_plot_t1`
  (Grain Field), `pasture_t1`, `charcoal_kiln_t1`, `smithy_t1`; `Building.Construct`
  attaches and initializes the `ProductionBuilding`.
- **Jobs.** New villager roles `Farmer, Herder, Charcoaler, Smith` (`VillagerJob`).
  Each walks to its matching production building and staffs the work slot (presence
  model, like the Guard); while present it registers as a live worker so the building's
  daily cycle counts it. `VillagerJobs.ProductionBuildingId` maps role â†’ building id.
- **Config.** All yields/cycles/recipes live in the **`EconomyConfig`** ScriptableObject
  (Resources path `EconomyConfig`, `CreateInstance` defaults â€” no asset file required):
  `productionRecipes`, the seasonal multipliers `springGrowthYield` (1.0),
  `summerGrowthYield` (1.5), `autumnGrowthYield` (2.0), `winterGrowthYield` (0.0), and
  `grainToFoodRatio`. Staffing radius / role speed live in `JobsConfig`.
- **Debug surface.** The **F2 EconomyDebugPanel** gains a *Production* section (each
  building's recipe kind, staffed workers, cycle %, completed cycles, and the current
  season) and shows the new resources in the stockpile list automatically. The `harvest`
  deposits also appear in the resource log tail.

Public API downstream tasks consume: `ProductionBuilding` (`Active`, `Recipe`,
`StaffedWorkers`/`IsStaffed`, `SetStaff`, `AddWorker`/`RemoveWorker`, `AdvanceDay`,
`CompletedCycles`, `CycleProgress01`), `EconomyConfig.RecipeFor` /
`SeasonalYieldMultiplier` / `GrainToFood`, the new `ResourceType` values, and
`VillagerJobs.IsProduction` / `ProductionBuildingId`. (P3-09 laws touch rations/night
labour via the ledger + jobs; P3-11 nightmares key off exploitation of these renewable
resources; P3-14 the spring-ship manifest consumes the stockpiled grain/meat/wool/tools
via the ledger and `GrainToFood`.)
